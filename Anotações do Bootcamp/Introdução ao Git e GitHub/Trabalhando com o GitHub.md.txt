# Trabalhando com o GitHub



## Nessa aula vamos

-  Criar um repositório remoto.
- Apontar o código da nossa máquina para um repositório remoto.
- E vamos aprender o comando utilizado para empurrar o código da nossa máquina para o código remoto da plataforma. 



### Mas antes...



Caso eu queira ver as configurações setadas no meu GIT eu posso utilizar o comando **"git config --list"**, e dessa forma eu verei uma lista de todas as configurações. 

E caso eu queira dar um unset nas configurações de email e username do meu git, basta eu utilizar o comando **"git config --global --unset user.email"**. E **"git config --global --unset user.name"**.





### Passos para empurrar os arquivos modificados no repositório remoto do GitHub



**1 -** Apontamos o nosso repositório, depois de realizar o commit. Dessa forma, adicionamos origem: **"git remote add origin https://link"**]



_Obs : O comando "git remote -v" possibilita com que eu veja quais são os  repositórios cadastrados._



**2 -** Posteriormente utilizamos o **"git push origin master (ou main)"**
